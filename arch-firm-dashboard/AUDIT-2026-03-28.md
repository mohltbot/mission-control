## What Was Broken (Before Tonight)

### Critical Issues Found
1. **System Apps Tracked as Work** 
   - `loginwindow` (Mac lock screen) counted as 45 hours of work
   - `UserNotificationCenter`, `SecurityAgent`, `Finder` all tracked
   - Result: 47 total hours logged, only 1 hour was actual work

2. **False Suspicious Activity Alerts**
   - 16,191 "suspicious" flags - all from system idle states
   - Real suspicious activities buried in noise
   - Productivity score: 3% (artificially low)

3. **Genesis AI Showing Garbage Data**
   - Top app: `loginwindow` with 45 hours
   - Employee productivity: 3% (should be ~70-90%)
   - Time wasters: System processes listed as distractions

### Root Cause
- Tracker recorded macOS system processes as employee activity
- SQL queries didn't filter system apps
- Idle detection too aggressive (2 min = "away from desk")

---

## What Was Fixed Tonight

### 1. Tracker Fixes (`desktop/src/tracker.ts` & `classifier.ts`)
- ✅ System processes filtered BEFORE recording (loginwindow, screensaver, etc.)
- ✅ Idle threshold: 2 min → 5 min (reduces spam)
- ✅ Never mark system processes as suspicious
- ✅ Relaxed thresholds: 15 min idle, 60 min same window

### 2. AI Analytics Fixes (`admin/server/routes/ai-routes.ts`)
- ✅ All SQL queries filter system apps using `COLLATE NOCASE`
- ✅ Added `category != 'break_idle' AND is_idle = 0` filters
- ✅ Productivity rankings exclude system idle time
- ✅ Suspicious activity counts exclude system processes

### 3. Server Fixes (`admin/server/index.ts`)
- ✅ Fixed static file path (was `dist/dist/client`, now `dist/client`)
- ✅ Added 50MB body parser limit (fixes JSON parse errors)
- ✅ Simplified path logic to always use correct directory

### 4. Pattern Detection Fixes (`admin/server/ai-analytics.ts`)
- ✅ Expanded `IGNORED_APPS` list for repetitive task detection

**Problem:** Server runs from `/opt/archtrack/arch-firm-dashboard/` (nested) but git repo is at `/opt/archtrack/` (parent)

**Impact:** 
- Git pull updates parent, server runs from nested
- Had to deploy fixes to nested directory separately
- Risk of version mismatch

**Recommendation:** 
- Consolidate to single directory structure
