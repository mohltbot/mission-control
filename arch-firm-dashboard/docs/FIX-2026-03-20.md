# ArchTrack Fix - March 20, 2026

## Problem
Desktop app showing "Sync failed for batch: Internal Server Error" when trying to sync 12,350 queued activities to DigitalOcean server at 165.227.78.107:3001.

## Root Cause
The Express.js body parser on the server had a default size limit (100kb) that was too small for large batches of activity data. When the desktop app tried to send batches of activities, the server rejected them with a 500 error.

## Solution

### Step 1: Fix Server Body Parser Limit
On the DigitalOcean server (via console):
```bash
nano /opt/archtrack/arch-firm-dashboard/admin/dist/server/index.js
```

Changed:
```javascript
app.use(express.json());
```

To:
```javascript
app.use(express.json({ limit: '50mb' }));
```

Saved with Ctrl+O, Enter, Ctrl+X

### Step 2: Restart Server
```bash
pm2 restart archtrack
```

### Step 3: Clear Corrupted Offline Queue
The old 12,350 queued activities had malformed JSON that was causing additional sync failures. Cleared the queue:
```bash
pkill -f "Electron"
rm ~/Library/Application\ Support/@archtrack/desktop/offline-queue.json 2>/dev/null
rm ~/Library/Application\ Support/archtrack-desktop/offline-queue.json 2>/dev/null
rm ~/Library/Application\ Support/Electron/offline-queue.json 2>/dev/null
```

### Step 4: Restart Desktop App
```bash
cd /Users/mohlt/.openclaw/workspace/arch-firm-dashboard/desktop
npm start
```

## Result
- ✅ Sync working: "✓ Synced 3 activities"
- ✅ Live Activity Feed updating in real-time
- ✅ Dashboard showing 56% Team Productivity (was 0%)
- ✅ WebSocket connected
- ❌ Lost 12,350 old queued activities (data corruption)

## Prevention
For future deployments, always set body parser limits:
```javascript
app.use(express.json({ limit: '50mb' }));
app.use(express.urlencoded({ limit: '50mb', extended: true }));
```
