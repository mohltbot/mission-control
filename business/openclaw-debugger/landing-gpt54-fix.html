Fix the GPT-5.4 Codex 401 Error in OpenClaw

ChatGPT Plus users: Your GPT-5.4 calls are failing with OAuth errors. Here's why — and how to fix it.

⚠️ The Problem

You're using GPT-5.4 via Codex OAuth in OpenClaw, but getting this error:

HTTP 401 Unauthorized
"Invalid authentication credentials"
Model falls back to GPT-5.3-codex silently

What's happening: OpenClaw is calling the wrong API endpoint for GPT-5.4 via Codex OAuth. The authentication flow succeeds, but subsequent API calls hit an endpoint that doesn't recognize the OAuth token.

⚡ Quick Fix (Copy-Paste)

Add this to your openclaw.yaml config:

# Force correct API endpoint for GPT-5.4 Codex
models:
  default:
    model: "openai/gpt-5.4"
    apiBase: "https://api.openai.com/v1"
    apiKey: "${OPENAI_API_KEY}"
    # Don't use Codex OAuth for GPT-5.4 — use direct API
    codexOAuth: false

Then restart: openclaw gateway restart

✅ The Complete Fix (3 Steps)

1

Switch to Direct API Authentication

The Codex OAuth flow has a bug with GPT-5.4 endpoint routing. Use direct API authentication instead:

# Get your API key from:
# https://platform.openai.com/api-keys

export OPENAI_API_KEY="sk-..."
2

Update Your Config

Edit ~/.openclaw/openclaw.yaml:

models:
  default:
    model: "openai/gpt-5.4"
    apiBase: "https://api.openai.com/v1"
    apiKey: "${OPENAI_API_KEY}"
    codexOAuth: false

# Optional: Keep Codex OAuth for other models
  codex:
    model: "openai/gpt-5.3-codex"
    codexOAuth: true
3

Verify the Fix

Test that GPT-5.4 is working:

openclaw gateway restart
openclaw doctor

# Check logs for successful GPT-5.4 calls:
openclaw gateway logs --follow | grep "gpt-5.4"

Frequently Asked Questions

Why does this only affect GPT-5.4?

GPT-5.4 uses a different API endpoint than GPT-5.3. The Codex OAuth flow authenticates correctly but then calls the wrong endpoint for GPT-5.4 specifically. GPT-5.3 via Codex OAuth works fine.

Will this be fixed in a future OpenClaw update?

Yes — this is a known issue tracked in GitHub issues. The workaround above will keep working even after the official fix.

Can I still use Codex OAuth for other models?

Yes! The config above shows how to keep Codex OAuth enabled for GPT-5.3 while using direct API for GPT-5.4.

How do I know if I'm affected?

Check your OpenClaw logs: openclaw gateway logs | grep "401". If you see 401 errors when calling GPT-5.4 but not other models, you're affected.

Is my data safe with direct API authentication?

Yes — direct API authentication is actually more secure than OAuth for automated systems. Your API key never leaves your machine (stored in environment variables).
