## What Is It?

**Claude Managed Agents** lets you build and deploy agents via Anthropic's developer console,
with Anthropic handling all the infrastructure. You define the agent behaviour; Anthropic
manages compute, scaling, and availability.

From Ben's Bites:
> "You can use Claude's developer console to build and deploy agents and let Anthropic handle
> the infra for it, vs building it yourself. For example, Notion is using managed agents to
> build a 'delegate tasks to Claude' feature."

Official announcement: https://anthropic.com/news/claude-managed-agents (verify link)
Anthropic engineering blog post on the implementation is also linked from the issue.

## Why It Matters for Mission Control

Currently, every Mission Control agent runs either:
- Locally in a Cowork session (ephemeral — dies when session ends)
- On a self-hosted VPS (DigitalOcean, requires maintenance)
- Via a cron job + Claude Code CLI (reliable but brittle on restart)

Claude Managed Agents offers a third path: **Anthropic-hosted, always-on agents** accessible
via the API, with no server to maintain.

### Key Benefits

| Concern | Current state | With Managed Agents |
|---|---|---|
| Uptime | Depends on DO droplet or local session | Anthropic SLA |
| Scaling | Manual | Automatic |
| Restarts | Manual or cron | Handled by platform |
| Model upgrades | Requires code redeploy | Swap model in console |
| Cost | Fixed VPS cost + tokens | Pay-per-use tokens only |

## Known Risks / Gotchas

- Managed Agents may restrict tool use (no arbitrary bash, no filesystem access)
- Pricing could exceed self-hosted at high volume (100+ agent runs/day)
- Vendor lock-in: tied to Anthropic's platform; migrating agents out is non-trivial
- Early product — stability and API surface may change
- Notion example suggests simple task-delegation; complex multi-step agents unconfirmed
