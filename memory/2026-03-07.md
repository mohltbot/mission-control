## New Lead Details

### 🔥 Hot Lead: GitHub #38336 — OAuth Recovery Loop Bug

**Issue:** https://github.com/openclaw/openclaw/issues/38336  
**Status:** Active, well-documented, affects many users  
**Pain Level:** HIGH — causes infinite safe-mode loops  
**User Frustration:** Multi-hour debugging sessions  

**The Problem:**
When OAuth tokens are renewed via `openclaw onboard`, the token is saved to auth-profiles.json (live copy) but NOT to auth-profiles.provisioned.json (golden copy). This causes safe-mode recovery to repeatedly fail because it reads from the stale golden copy.

**Why This Is Hot:**
- User explicitly mentions "multi-hour safe-mode debugging session"
- Clear workaround provided but not obvious
- Affects anyone using OAuth renewal
- High visibility issue (good for credibility if you help)

**Draft Action:** Comment with simplified workaround + offer to review their config
